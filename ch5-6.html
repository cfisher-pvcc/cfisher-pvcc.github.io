Von Neumann computer architecture, 4 major subsystems:
  1. Memory
  2. Control Unit
  3. Arithmetic/Logic Unit (ALU)
  4. Input-Output
Von Neumann computer architecture, 3 characteristics:
  1. Four major subsystem called memory, input/output, the arithmetic/logic unit (ALU), and the control unit. The ALU and the control unit are often bundled together in what is called the central processing unit or CPU.
  2. The stored program concept, in which the instructions to be executed by the computer are represented as binary values and stores in memory.
  3. The sequential executions of instructions, in which one instruction at a time is fetched from memory and passed to the control unit, where it is decoded and executed.
Computer Archetecture Terms:
  1. RAM:RAM is a common computing acronym that stands for random-access memory. In essence, RAM is your computer or laptop's short-term memory. It's where the data is stored that your computer processor needs to run your applications and open your files.
  2. Cache:A hardware or software component embedded in an application or device memory that automatically and temporarily stores data consumed by the user to reduce the data retrieval time the next time the application or device is accessed.
  3. Principle of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. It will access that same instruction or piece of data in the very near future.
    2. It will soon access the instructions or data that are located near that piece of data, where "near" means an address whose numerical value is close to this one.
  4. ROM: A special type of random access memory into which information has been prerecorded during manufacture. This information cannot be modified or removed, only fetched.
  5. Memory Cell Size:Memory cells can be of different sizes: 8 bits, 16 bits, 32 bits, 64 bits. Different sized cells are used to store different types of data.
  6. Memory Cell Address: a unique identifier used by a device or CPU for data tracking. This binary address is defined by an ordered and finite sequence allowing the CPU to track the location of each memory byte.
  7. Maximum Memory Size: the maximum total RAM that the processor will address. It is also known as the value 2^N
  8. Register: A special high-speed storage cell that holds the operands of an arithmetic operation and that, when the operation is complete, holds its results.
  9. Fetch:The retrieval of data by a software program, script, or hardware device.
  10. Store:A sequence of machine language instructions stores as binary values in memory.
  11. Memory Access Time:how long it takes for a character in RAM to be transferred to or from the CPU.
  12. DASD: Direct access storage devices (DASDs) are fixed or removable storage devices.
  13. Arithmetic and Logic Unit (ALU): The subsystem that perfroms such mathmematical and logical operations such as addition, subrtaction, and comparison for eqaulity.
  14. Control Unit: The control unit fetches from memory the next instruction to executed, decode it and execute it by issuing the appropriate command to the ALU.

Four things the interface between the computer and the hardware does:
  1. Hides from the user the mesy details of the underlying hardware.
  2. Presents information about what is happening in a way that does not require in-depth knowledge of the internal structure of the system.
  3. Allows easy user access to the resources available on this computer
  4. Prevents accidental or intentional damage to hardware, programs and data
Responsibilities of systems software:
  1. Hides the complex and unimportant (to the user) details of the internal structure of the Von Neumann architecture.
  2. Presents important information to the user in a way that is easy to uderstand.
  3. Allows the user to access machine resources in a simple and effivient way.
  4. Provides a secure and safe environment in which to operate.
Terms:
  1. Systems Software: Is a collection of computer programs that manage the resources of a computer and facilities access to those resources.
  2. User Interface: All modern operating systems provide a powerful graphical user interface (GUI) that gives the user intuitive visual overview as well as graphical control of the capabilities and services of the computer. Control of the GUI is typically done with keystrokes, mouse clicks, finger taps, voice activation, or biometrics scans such as fingerprints.
  3. Language Services: These programs, called assemblers, compliers, and interpreters, allow you to write programs in a high-level, user-oriented language rather than machine language and to execute these programs easily and efficiently. They often include components such as text editors and debuggers.
  4. Memory Managers: These programs llocate memory space for programs and data and retrieve this memory space when it is no longer needed.
  5. Information Managers: These programs handle the organization, storage, and retrieval of information on storage devices such as the hard drives, DVDâ€™s, flash drivers, and tapes as well as information stored remotely in data centers. They allow you to organize your information in an efficient high hierarchical manner, using directories, folders, and files and to keep your personal data safe from accidental or intentional misuse.
  6. i/o Managers:These software packages allow you to easily and efficiently use the many different types of input and output devices that exist on a modern computer system.
  7. Schedualer:This system program keeps a list of programs ready to run on the processor, and it selects the one that will execute next. The scheduler allows you to have several different programs active at a single time, for instance, surfing the web while you are waiting for a file to finish printing.
  8. Utilities: These collections of library routines provide a wide range of useful services either to a user or to other system routines. Text editors, online help routines, image and sound applications, and control panels are examples of utility routines. Sometimes these utilities are organized into collections and called program libraries.
  9. Assembly Language: a low-level symbolic code converted by an assembler.
Functions of an Operating System:
  1. User interfacce managment.
  2. Control of access to the system and to data files.
  3. Program scheduling and activiation.
  4. Efficient resource allocation.
  5. Deadlock detection and error detetion.
