All about Github

  • Q. What is a version control system?
  • A. It is the practice of tracking and managing changes to software code.
  • Q. What is Git?
  • A. Git is a free and open source distributed version control system.
  • Q. What is GitHub?
  • A. GitHub is a git repository hosting service that provides a web-based graphical interface.
  • Q. What is the difference between Git and GitHub?
  • A. Git is a version control system that lets you manage and keep track of your source code history. GitHub is a cloud-based hosting service that lets you manage Git repositories.
  • Q. Who started GitHub and why was it started?
  • A. GitHub was started by Tom Preston-Werner, Chris Wanstrath,PJ Hyett and Scott Chacon. Wanstrath and Preston-Werner needed GitHub for their own work, and so they developed their tool out of necessity. The two men quickly identified a major problem in their work—forking code branches and collaborating on programming projects—and devised a solution that met their needs.
  • Q. What company owns it now?
  • A. Microsoft
  • Q. How much does a Github account cost?
  • A. A basic account for individuals and organizations are free. An advanced collaberation is currently $44 and a security, compliance and flexible deployment account is $231.
  • Q. What is the Octocat?
  • A. The Octocat is the mascot of the source-code hosting service GitHub.

Git/GitHub Terms:

  • Repository: contains all of your project's files and each file's revision history.
  • Commit:A commit, or "revision", is an individual change to a file (or set of files).
  • Fork:is a new repository that shares code and visibility settings with the original “upstream” repository.
  • Push: used to upload local repository content to a remote repository.
  • Pull Requests:let you tell others about changes you've pushed to a branch in a repository on GitHub.
  • Work Flows:a configurable automated process that will run one or more jobs.
  • Issues:a lightweight issue-tracking system.
  • Raw Button:you can view or copy the raw content of a file without any styling.
  • Blame Button:The blame command is a Git feature, designed to help you determine who made changes to a file.
